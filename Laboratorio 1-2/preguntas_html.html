Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

El Internet es la red física de dispositivos interconectados los unos con los otros, mientras que la World WIde Web es el conjunto de páginas y software que corre entre esas conecciones.

¿Cuáles son las partes de un URL?

El protocolo, el dominio, el camino y los parámetros.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Es un método que sirve para obtener información del servidor web. No cambia el estado de los datos.
  • Head: También sirve para obtener información del servidor, sin embargo, este sólo pide el encabezado de esta forma se puede saber si está disponible antes de hacer la solicitudcon el GET.
  • POST: Se usa para enviar información al servidor.
  • PUT: Sirve para alterar la información del cuerpo.
  • PATCH: Modifica parcialmente la página.
  • DELETE: Borra una parte específica de la página.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

HTTPS, debido a que el protocolo está encriptado y permite que se mantenga la seguridad de la información.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Get debido a que solicitas información a través de él

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la página cargó correctamente, no ocurrió ningún error, los códigos que comienzan en 2 y están seguidos de dos digitos indican que ha sucedido algo correctamente

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No siempre, el error puede suceder porque el usuario ingresó la URL incorrectamente, la página dejó de existir o el servidor dejó de funcionar, lo cual no está necesariamente en las manos del desarrollador para arreglar.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, debido a que este error representa que hubo una falla interna en el servidor, por lo que el o los desarrolladores son los encargados de revisarlo.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que un elemento o elementos se ha vuelto obsoleto, y por lo tanto, ya no hay soporte. Aling, era usado para especificar la posición de un elemento; alink, era usado para seleccionar el color de un link y bgcolor, especificaba el color del fondo.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • HTML4 no soporta multimedia, mientras que HTML5 sí.
  • HTML4 utiliza la etiqueta applets que fueron sustituidos en HTML5 por la etiqueta object
  • HTML4 carece de web sockets, mientras que HTML5 los poseé y permite una comunicación bidireccional y full-duplex entre el navegador y el servidor

¿Qué componentes de estructura y estilo tiene una tabla?

Columnas y filas que pueden unirse en una sola dependiendo de cómo se acomoden los elementos entre las funciones tr y th, los número de columnas y filas aumentan dependiendo del número de los mismos que se agregen en el código.
